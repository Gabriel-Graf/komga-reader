**Stand:** 2026-06-12 · **Status:** Design (noch nicht gebaut) · **Sub-Projekt I1** der
Roadmap `docs/superpowers/specs/2026-06-12-complete-ui-modularity-roadmap.md` (neuer Zweig: Icons)

> **Self-contained:** Frische Session ohne Gesprächskontext kann das umsetzen. Vor dem Bauen lesen:
> `komga-plugins`-Skill (Capability-/Pack-Philosophie), `eink-design-language.md` (Icon-System: Lucide via
> `AppIcons`-Registry, E-Ink-Stroke). **Vorbild für die Pack-Idee:** die UI-Slot-Regionen + das Theme-Pack
> (`UiPack`) — aber I1 nutzt **bewusst NICHT** `CompositionLocal` (Begründung §2).

## 1. Ziel & User-Entscheidungen (2026-06-12)

Die Icons austauschbar machen: ein **Icon-Pack** kann die Glyphen hinter den semantischen `AppIcons.*`-
Namen app-weit ersetzen, ohne Aufruf-Stellen anzufassen.

**User-Entscheidungen:**
- **Mechanismus:** eine **eigene, prozess-globale `IconPack`-Registry** (kein `CompositionLocal`), an die
  `AppIcons` delegiert; typsicher über ein `IconKey`-Enum; Default = die heutige Lucide-Map mit Fallback.
- **Import-Scope jetzt:** **nur die In-tree-Infra** (Indirektion + `IconKey` + ein Built-in-Alternativ-Set
  als Swap-Beweis). Der **externe Icon-Pack als APK** kommt **später** zusammen mit dem externen UI-Pack-
  Lader (L1/L2, TOFU/ABI wie plugin-host). Runtime-SVG-Import ist **nicht** Teil von I1.

## 2. Warum globale Registry statt `CompositionLocal`

`AppIcons.Home` wird an **zwei** Arten von Stellen gelesen:
1. in `@Composable`-Code (`Icon(AppIcons.Home, …)`),
2. **außerhalb** von Composition — in **Datenklassen-Feld-Initialisierern** (`SettingsSection(icon =
   AppIcons.Connection)`, `ShellDestination(icon = …)`, `EinkBottomBar`-Items) und **Default-Argumenten**
   (`actionIcon: ImageVector = AppIcons.Search`).

`CompositionLocal` ist **nur in `@Composable`-Scope** lesbar → es würde die Nicht-Composable-Call-Sites
brechen. Darum delegiert `AppIcons` an eine **prozess-globale** Registry (`ActiveIconPack`), die in **allen**
Kontexten funktioniert. Das passt auch konzeptionell: Icons sind ein **app-weiter Look** (wie das Theme),
kein per-Subtree-Zustand. (Live-Umschalten zur Laufzeit ist begrenzt — die Datenklassen-Felder lesen beim
Bau; das ist für I1 ok, der Pack wird app-weit gesetzt. Reaktives Live-Switching ist Soll mit dem späteren
Lader.)

## 3. Ausgangslage (Ist, verifiziert)

- **`app/.../ui/icons/AppIcons.kt`** (`object AppIcons`, Z. 14-60): 41 semantische `val X: ImageVector
  get() = LucideIcons.Y`. **Das `get()` ist schon dynamisch** — die einzige Änderung ist, *woher* es liest.
- **`app/.../ui/icons/LucideIcons.kt`** — die generierten Glyphen (aus `tools/icons`). **Unverändert.**
- **Nutzung:** 29 Dateien, ~117 Stellen `AppIcons.*`; **0 Bypässe** (keine Material-Icons, kein
  `painterResource` für Icons). Alle `icon: ImageVector`-Datenfelder zeigen auf `AppIcons.*`.
- **`tools/icons`** (`icon-set.mjs` → `generate.mjs` → `LucideIcons.kt`): der Glyph-Generator. **Unverändert**
  (neue Glyphen weiterhin compile-time generiert).
- **`UiSlots.kt`/`UiPack.kt`** — Pack-Muster als Referenz (aber I1 ist global, nicht CompositionLocal).

## 4. Design

Neue Datei `app/src/main/kotlin/com/komgareader/app/ui/icons/IconPack.kt`.

### 4.1 `IconKey` (typsicherer Schlüssel)

Ein Enum mit **genau einem Eintrag pro `AppIcons`-Property** (41), gleiche Namen:
```kotlin
enum class IconKey {
    Close, Back, Forward, Check, Plus, Minus, ChevronRight, ChevronDown, ChevronUp, Home, Search,
    Refresh, Edit, Settings, Delete, Download, Local, Cloud, Info, Filter, Overflow, Stop, GridView,
    LargeGridView, ListView, Bookmark, BookmarkFilled, Library, Groups, Plugins, Contrast, Palette,
    Reader, Language, Connection, ReaderMode, PanelMode, Typography, TableOfContents, AlignLeft, AlignJustify,
}
```

### 4.2 `IconPack`-Vertrag + Default (die eine Lucide-Map)

```kotlin
/** Ein Icon-Pack liefert Glyphen für die semantischen [IconKey]s. `null` = diesen Key nicht überschreiben
 *  → Fallback auf das Default-Pack (analog StubSource/Slot-Fallback). */
fun interface IconPack {
    fun resolve(key: IconKey): ImageVector?
}

/** Das mitgelieferte Default-Pack: die heutige Lucide-Zuordnung — die EINE Map (kein Duplikat in AppIcons). */
object DefaultIconPack : IconPack {
    override fun resolve(key: IconKey): ImageVector = when (key) {
        IconKey.Close -> LucideIcons.X
        IconKey.Back -> LucideIcons.ArrowLeft
        // … alle 41, 1:1 aus dem heutigen AppIcons.kt übernommen …
        IconKey.AlignJustify -> LucideIcons.AlignJustify
    }
}
```
> Die Lucide-Map wandert **aus `AppIcons.kt` hierher** (DefaultIconPack ist jetzt die SSOT der Zuordnung).
> `DefaultIconPack.resolve` ist **non-null** (deckt alle Keys); der `IconPack`-Vertrag erlaubt aber `null`
> (Teil-Packs überschreiben nur einige).

### 4.3 `ActiveIconPack` (globaler, austauschbarer Halter)

```kotlin
/** App-weiter, austauschbarer Icon-Pack-Halter. [current] wird app-weit gesetzt (z. B. am Start aus einer
 *  Einstellung / später vom Pack-Lader). [resolve] fällt pro Key sauber auf [DefaultIconPack] zurück, sodass
 *  ein Teil-Pack nie ein Loch lässt. */
object ActiveIconPack {
    @Volatile var current: IconPack = DefaultIconPack
    fun resolve(key: IconKey): ImageVector = current.resolve(key) ?: DefaultIconPack.resolve(key)
}
```

### 4.4 `AppIcons` delegiert (Call-Sites unverändert)

```kotlin
object AppIcons {
    val Close: ImageVector get() = ActiveIconPack.resolve(IconKey.Close)
    val Back: ImageVector get() = ActiveIconPack.resolve(IconKey.Back)
    // … alle 41 …
    val AlignJustify: ImageVector get() = ActiveIconPack.resolve(IconKey.AlignJustify)
}
```
**Keine** der ~117 Call-Sites ändert sich (`AppIcons.Home` bleibt der Zugriff). KDoc von `AppIcons`
aktualisieren: zeigt jetzt über `ActiveIconPack`/`IconKey` auf das aktive Pack (Default = `DefaultIconPack`).

### 4.5 E-Ink-Invarianten (host-erzwungen)

Der E-Ink-**Stroke** (2.5f, `StrokeCap/Join.Round`) lebt in `LucideIcons.kt`/dem Generator — ein Pack, das
Lucide-Glyphen liefert, erbt ihn. Liefert ein Pack eigene `ImageVector`s, ist deren Stroke Pack-Sache; die
**Monochrom-/Tint-Politik** bleibt an den Call-Sites (`Icon(tint = …)`) host-erzwungen, nicht im Pack.

## 5. Swap-Beweis

- **Primär (Test, §6):** ein `AlternativeIconPack` (überschreibt z. B. `Home`→`LucideIcons.Library`,
  `Settings`→ anderer Glyph; gibt für andere Keys `null` zurück) — der Test setzt `ActiveIconPack.current`
  darauf und prüft, dass `AppIcons.Home` den Alt-Glyph liefert **und** ein nicht-überschriebener Key auf den
  Default zurückfällt.
- **Optional (Debug-Preview):** `app/src/debug/.../ui/icons/IconPackPreview.kt` — setzt vor dem `@Preview`
  ein Alt-Pack und rendert ein kleines Icon-Raster (zeigt überschriebene vs. Default-Glyphen). Nur Debug.

## 6. Tests

Neue `app/src/test/.../ui/icons/IconPackTest.kt` (pure JVM):
- `Default-Pack deckt jeden IconKey ab` → für **alle** `IconKey.values()` ist `DefaultIconPack.resolve(key)`
  non-null (kein vergessener Key — fängt eine später hinzugefügte Enum-Konstante ohne Mapping).
- `fehlende Überschreibung fällt auf das Default-Pack zurück` → Alt-Pack gibt `null` → `ActiveIconPack.resolve`
  liefert den Default-Glyph (`assertSame` auf den Lucide-`ImageVector`).
- `geliefertes Pack überschreibt den Default` → Alt-Pack mappt `Home` → ein anderer Glyph; `ActiveIconPack.
  resolve(IconKey.Home)` == dieser Glyph; `AppIcons.Home` == dieser Glyph.
- **`@AfterEach`/try-finally:** `ActiveIconPack.current = DefaultIconPack` zurücksetzen (globaler Zustand —
  Test-Isolation). Echte Umlaute in Testnamen.

## 7. Akzeptanz

- `IconKey` (41, = `AppIcons`-Properties), `IconPack` (fun interface, nullbar), `DefaultIconPack` (die eine
  Lucide-Map, aus `AppIcons.kt` hierher gezogen), `ActiveIconPack` (globaler Halter + Fallback). `AppIcons.*`
  delegiert; **keine** der ~117 Call-Sites geändert; `LucideIcons.kt`/`tools/icons` unverändert.
- Pure Tests grün (Vollständigkeit + Fallback + Override). Compile grün. Swap-Preview kompiliert (falls
  gebaut). E2E: App startet, ein paar Screens zeigen **unveränderte** Default-Icons (Bibliothek/Settings/
  Reader-Chrome) — verhaltens-/pixelgleich (Default-Pfad).
- `eink-design-language.md` (Icon-System: jetzt über `IconKey`/`ActiveIconPack` austauschbar, Default =
  `DefaultIconPack`) + `big-picture-and-goals.md` (Modulare-UI-Tabelle: Icon-Pack-Infra gebaut; externer
  Icon-Pack-APK = Soll mit L1/L2) + Memory-Roadmap im selben Commit nachgezogen.

## 8. Nicht in I1 (YAGNI)

**Kein** externer Pack-Lader / APK-Icon-Pack (→ L1/L2). **Kein** Runtime-SVG-Import. **Keine** Nutzer-
Einstellung zum Umschalten (der Halter ist da, eine Settings-UI kommt mit dem Lader). **Keine** Änderung an
`LucideIcons.kt`, `tools/icons`, oder den Call-Sites. **Kein** `CompositionLocal` (bewusst, §2). **Keine**
reaktive Live-Umschaltung (Soll mit dem Lader). Icons bleiben getrennt vom Theme-`UiPack` (eigene Naht).
