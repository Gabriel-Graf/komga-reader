Architektur — erweitert über die Naht-A-Schnittstelle (nicht Komga-spezifisch)

  • Series (domain) bekommt summary, status, genres — generische Metadaten, die jede Quelle füllen kann.
  • Book (domain) bekommt summary, number (für Fallback-Beschreibung + Kapitelnummer).
  • BrowsableSource erhält suspend fun seriesDetail(remoteId): Series? — jede Quelle implementiert: Komga (GET series/{id}), OPDS (aus Feed bzw. null).
  • Komga: DTOs um summary/genres/publisher erweitert, Mapper mappt, neuer Endpoint. Andere Server hängen sich rein über das Interface ein — kein Kern-Umbau.
